## Security policy

### Supported versions

Security fixes are provided for the **latest** version on the default branch.

### Reporting a vulnerability

- **Preferred**: Use GitHub **Private Vulnerability Reporting** / **Security Advisories** for this repository.
- **If that’s not available**: Open an issue with **no sensitive details**, and request a secure channel from maintainers.

Please include:

- A clear description of the impact
- Reproduction steps / proof of concept
- Affected versions/commit (if known)
- Any suggested remediation

### Disclosure process

We aim to acknowledge reports promptly and coordinate a fix and release before public disclosure when feasible.
